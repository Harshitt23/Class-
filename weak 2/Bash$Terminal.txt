chmod ‚Äî Change File Permissions || Modifies who can read (r), write (w), or execute (x) a file or directory.
Stands for ‚Äúchange mode.‚Äù

üìò Syntax:
chmod [permissions] [filename]


üìò Examples:
Grant execute permission to a file:

chmod +x script.sh


‚û°Ô∏è Now script.sh can be executed as a program.

Set specific numeric permissions:

chmod 755 app.js


‚û°Ô∏è Means:

Owner: read, write, execute
Group: read, execute
Others: read, execute

Remove write permission from everyone:
chmod a-w file.txt


üìò Common Permission Modes:
777 ‚Üí Full access for everyone (‚ö†Ô∏è unsafe)
755 ‚Üí Standard for executables
644 ‚Üí Standard for text/code files

head ‚Äî Display Beginning of File || Shows the first few lines (default 10) of a file.

head filename.txt
head -n 5 filename.txt   # first 5 lines
head -c 20 filename.txt  # first 20 bytes

-------------------------------------------------------------------------------------------------------------------------------------

tail ‚Äî Display End of File || Shows the last few lines (default 10) of a file. Commonly used to monitor log files.

tail filename.txt
tail -n 5 filename.txt   # last 5 lines
tail -f logfile.txt      # follow live updates
